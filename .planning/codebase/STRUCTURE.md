## Directory Purposes

**src/components/:**
- Purpose: UI components composing pages and features
- Contains: Custom components, shadcn/ui primitives, step components for meditation workflow
- Key files: `MeditationPlayer.tsx`, `MoodCheck.tsx`, `SoundPicker.tsx` form meditation workflow
- Notes: `ui/` subdirectory is auto-generated by shadcn CLI; do not edit manually

**src/pages/:**
- Purpose: Route page components mounted by React Router
- Contains: Four main pages (Index, TrackerPage, AnalyticsPage, SettingsPage) plus NotFound fallback
- Key files: `Index.tsx` is the main meditation interface; others display journal, analytics, settings
- Pattern: Each page is a page-level component that imports sub-components

**src/lib/:**
- Purpose: Core application logic not tied to React components
- Contains: Zustand store, Web Audio API wrapper, analytics functions, utilities
- Key files:
  - `meditation-store.ts`: Zustand store, mood configs, quote data, localStorage persistence
  - `ambient-engine.ts`: AmbientEngine and MeditationDrone classes for audio
  - `analytics.ts`: Event tracking functions wrapping Vercel Analytics API

**src/hooks/:**
- Purpose: Reusable React hook utilities
- Contains: `use-pull-to-refresh`, `use-mobile`, `use-toast` from shadcn UI
- Pattern: Each hook exports a single default hook function

**src/integrations/supabase/:**
- Purpose: Third-party service configuration
- Contains: Supabase client factory and TypeScript type definitions
- Current state: Configured but not actively used; available for future features

**src/test/:**
- Purpose: Unit and integration test files
- Contains: Vitest setup configuration and example tests
- Pattern: Test files co-located by feature or in this central directory

**public/sounds/:**
- Purpose: Audio files for meditation ambient sounds
- Contains: 9 pre-recorded ambient sound MP3 files (~30-60 seconds each, looped)
- Mapping: File names match keys in SOUND_FILES object in `ambient-engine.ts`
- Attribution: Free Sound Library with CC BY 4.0 license (credited in BottomNav)

**public/images/:**
- Purpose: Illustration assets displayed during meditation sessions
- Contains: Sound-specific meditation cat illustrations
- Mapping: One image per sound type (meditation-cat-{sound-type}.png)
- Pattern: Dynamically loaded based on selected sound in MeditationPlayer

**public/videos/:**
- Purpose: Hero animations
- Contains: Hero video for home page background

**Entry Points:**

**Testing:**

## Naming Conventions

**Files:**
- Components: PascalCase (e.g., `MeditationPlayer.tsx`, `MoodCheck.tsx`)
- Pages: PascalCase with "Page" suffix (e.g., `TrackerPage.tsx`, `AnalyticsPage.tsx`)
- Utilities/hooks: camelCase (e.g., `ambient-engine.ts`, `use-pull-to-refresh.ts`)
- Types: PascalCase in meditation-store.ts (e.g., `PreMood`, `MoodEntry`)

**Directories:**
- Feature directories: kebab-case (e.g., `src/components/`, `src/integrations/supabase/`)
- No src subfolders use camelCase; all are lowercase or kebab-case

**Functions/Variables:**
- React components: PascalCase export
- Utility functions: camelCase (e.g., `trackPageView()`, `getRandomQuote()`)
- Constants: camelCase (e.g., `SOUND_FILES`, `BREATH_PHASES`)
- Types/Interfaces: PascalCase (e.g., `MeditationState`, `MoodEntry`)

**Imports:**
- Path alias `@/` always used for src/ imports: `import X from '@/lib/...'`
- No relative path imports (../../../) used in codebase

## Where to Add New Code

**New Meditation Sound:**
1. Add MP3 file to `public/sounds/{name}.mp3`
2. Update `SOUND_FILES` object in `src/lib/ambient-engine.ts`
3. Update `SoundType` union type in `src/lib/meditation-store.ts`
4. Add config entry to `soundConfig` object in `src/lib/meditation-store.ts`
5. Add illustration to `public/images/meditation-cat-{name}.png`
6. Update `SoundPicker` component render branches or use dynamic rendering pattern

**New Mood Type:**
1. Update `PreMood` or `PostMood` type union in `src/lib/meditation-store.ts`
2. Add config to `preMoodConfig` or `postMoodConfig` object with icon, label, color
3. Update `MoodCheck` component to render new mood option
4. Update mood-to-value mapping in `postMoodToValue` or `preMoodToValue`

**New Inspirational Quote:**
1. Add quote object to `stoicQuotes` array in `src/lib/meditation-store.ts`

**New Page:**
1. Create component in `src/pages/{PageName}.tsx`
2. Add route to `App.tsx` Routes definition
3. Import page component in `App.tsx`
4. Add navigation link in `BottomNav.tsx` if it should appear in footer nav

**New Component:**
1. Create file in `src/components/{ComponentName}.tsx` (PascalCase)
2. If it's a shadcn/ui primitive: run `npx shadcn@latest add {component-name}`
3. If it's custom: import dependencies and export as default

**New Hook:**
1. Create file in `src/hooks/use-{name}.ts` (kebab-case filename)
2. Export hook function as default
3. Document props and return type

**Analytics Event:**
1. Add tracking function to `src/lib/analytics.ts`
2. Call function from component using Vercel Analytics SDK

## Special Directories

**src/components/ui/:**
- Purpose: shadcn/ui primitive components (auto-generated)
- Generated: Yes — created by `npx shadcn@latest add [component]`
- Committed: Yes — checked into version control
- Edit Policy: Do not manually edit; regenerate if needed via CLI
- Contains: Accordion, AlertDialog, Button, Card, Dialog, Slider, Tabs, Toast, and 50+ others

**public/:**
- Purpose: Static assets served without bundling
- Generated: No
- Committed: Yes
- Contains: Sounds, images, videos
- Accessible via: `/sounds/`, `/images/`, `/videos/` URLs

**.planning/codebase/:**
- Purpose: Architecture documentation for Claude Code reference
- Generated: No — written manually during analysis
- Committed: Yes
- Contains: ARCHITECTURE.md, STRUCTURE.md, STACK.md, INTEGRATIONS.md, CONVENTIONS.md, TESTING.md, CONCERNS.md

**supabase/:**
- Purpose: Supabase local development configuration
- Generated: Yes — by Supabase CLI
- Committed: No — in .gitignore
- Contains: Local database migrations and types

**node_modules/:**
- Purpose: Installed npm dependencies
- Generated: Yes
- Committed: No
- Note: Install with `npm install` or `bun install`
